1. SYSTEM OVERVIEW
--------------------------------------------------------------------------------
The Deterministic GRC Control Validator is an enterprise-grade, privacy-preserving 
architecture designed to evaluate technical engineering standards, System Security 
Plans (SSPs), and baseline configurations against codified regulatory frameworks 
(ISO 27001, ISO 42001, NIST 800-53, EU AI Act).

Operating under a strict defense-in-depth security paradigm, the engine demonstrates 
how to leverage Large Language Models (LLMs) for complex technical gap analysis 
without exposing the application to prompt injection breakouts, data exfiltration, 
or Local File Inclusion (LFI) vulnerabilities.

2. CORE ARCHITECTURE & SECURE-BY-DESIGN SPECIFICATIONS
--------------------------------------------------------------------------------
[A] Zero-Trust UI & Data Routing (app.py)
  - Path Traversal Fencing: Telemetry log paths are strictly hardcoded in the 
    backend, completely removing Arbitrary File Write vulnerabilities.
  - Intelligent Database Routing: User UI intent is mapped to immutable backend 
    constants. The frontend has zero direct manipulation access to the underlying 
    file system or vector paths, preventing LFI exploitation.

[B] Adversarial Input Firewall & Data Sanitization (sanitize.py)
  - Prompt Injection Defense: Implements a regex-based firewall that actively 
    scans for and intercepts malicious structural overrides (e.g., "ignore 
    previous instructions") before they reach the LLM execution context.
  - PII & Proprietary Masking: Automatically detects and redacts personal 
    identifiable information (PII) and proprietary corporate markers.

[C] Secure Orchestration & Context Fencing (orchestrate.py)
  - XML Context Fencing: Wraps untrusted user standards inside hard XML tags 
    (<document_context>) to strictly separate system-level compliance instructions 
    from user-provided data.

[D] Deterministic Output Gate / Circuit Breaker
  - Pydantic Schema Validation: Replaces standard LLM text generation with a 
    strict, type-enforced schema pipeline. A custom healing layer patches minor 
    LLM drift, while the circuit breaker actively drops any response that 
    hallucinates data or misses mandatory enterprise reporting keys.

[E] Dynamic Regulatory Overlay Engine (transform.py)
  - Multi-Framework Crosswalking: Dynamically maps technical engineering controls 
    (like NIST 800-53) directly to overarching statutory laws (e.g., GDPR, EU AI Act).
  - Weighted Semantic Router: Routes technical text to the correct compliance tracks 
    using a custom scoring matrix based on primary framework identifiers, semantic 
    hooks, and context token intersection.

3. INTENDED OPERATIONAL USE CASES
--------------------------------------------------------------------------------
The system is explicitly engineered for the following operational workflows:
* System Security Plan (SSP) Validation: Rapid parsing of technical documentation 
  to discover control deficiencies prior to formal SOC 2 or ISO certification audits.
* DevSecOps Configuration Auditing: Acting as a compliance linter to ensure written 
  infrastructure baselines match regulatory expectations.
* AI Architecture Alignment: Cross-referencing ML engineering specifications against 
  emerging legal requirements (ISO 42001, Brazilian AI Act).

*Note: This engine evaluates quantifiable engineering standards. High-level corporate 
policies lacking technical parameters will intentionally generate low compliance 
scores to highlight governance abstraction gaps.*
